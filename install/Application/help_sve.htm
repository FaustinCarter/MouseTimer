Att använda Data Replicator 3

Klicka på följande huvudrubriker för att lära känna Data Replicator 3:

  • Säkerhetskopiering av PC-data
  • Återställning av säkerhetskopierade filer
  • Avancerade inställningar
  • Loggfil

Säkerhetskopiering av PC-data

Vänligen följ nedanstående steg för att göra inställningar för säkerhetskopiering. Du kan även gå till Avancerade inställningar för att bekanta dig med de övriga inställningarna som kan göra din säkerhetskopiering av PC-data personlig.

  1. Välj ett mål för säkerhetskopian

    2. Du kan välja att säkerhetskopiera din PC-data till en Synology-server eller till en annan plats. Efter att säkerhetskopieringen har utförts korrekt kommer du att hitta de säkerhetskopierade filerna i den mapp som du angivit.

    1.1 Säkerhetskopiera till en Synology-server

    1. Klicka på Välj.
    2. Välj Synology-server och klicka på OK.
    3. I nästa fönster klickar du på den Synology-server där den delade mappen för säkerhetskopieringsfiler finns. Klicka sedan på Nästa.
    4. Ange det användarnamn och lösenord som kan läsa och skriva till den delade mapp du just valde, och klicka sedan Nästa.
    5. Klicka på den delade mapp du vill använda för säkerhetskopieringen och klicka på Slutför.

    1.2 Säkerhetskopiera till annan plats

    1. Klicka på Välj.
    2. Välj Annan plats och klicka sedan på Bläddra för att lokalisera målet för säkerhetskopian.
    3. Klicka på OK för att slutföra.

  2. Markera de mappar du vill säkerhetskopiera

  3. Välj ett säkerhetskopieringsläge

    4. Data Replicator 3 erbjuder 3 säkerhetskopieringslägen för största bekvämlighet. Dessa lägen är: Omedelbar, Synkroniserad och Schema.

    3.1 Omedelbar

         Genom att klicka på knappen Direkt säkerhetskopieras samtliga filer i markerad(e) lokal(a) mapp(ar) genast till målet för säkerhetskopian.

    3.2 Synkroniserad

    • Synkronisera data nu och börja övervaka

      • Klicka på knappen Synk och Data Replicator 3 säkerhetskopierar samtliga filer i markerad(e) lokal(a) mapp(ar) genast till målet för säkerhetskopian. Därefter påbörjar Data Replicator 3 övervakning av denna/dessa mappar. Om du gör ändringar säkerhetskopierar Data Replicator 3 bara de filer som har ändrats.

      Obs!

      1. Sedan Data Replicator 3 gått in i övervakningsläge fortsätter den arbeta till dess du avbryter den.
      2. Tidsintervallen mellan mailsynkroniseringarna gäller från och med slutförandet av mailsynkroniseringen till nästa synkroniseringsstart. Uppgiften kan senareläggas när Data Replicator 3 utför synkronisering av andra filer.

    • Avbryta övervakning

      • För att avbryta övervakningen klickar du på Avbryt Synk i programfönstret till Data Replicator 3. Om programmet är minimerat till systemverktygsfältet dubbelklickar du på programikonen för Data Replicator 3 för att öppna det.

    3.3 Schema

    • Lägg till schema

      • Data Replicator 3 kan ställas in för att automatiskt utföra regelbundna säkerhetskopieringar vid en specificerad tidpunkt. Gör som följande:

      1. Klicka på Schema.
      2. Ange period och starttid för den schemalagda säkerhetskopieringen. Klicka därefter OK.
      3. Om du vill att schemalagd säkerhetskopiering skall utföras under tider då du är utloggad, var vänlig ange ditt användarnamn och lösenord. Klicka sedan på OK.
        Om du inte vill utföra säkerhetskopiering under tider då du är utloggad kan du klicka på Hoppa över.

      Obs!

      1. Lösenord får ej vara tomt då du vill säkerhetskopiera under tider då du är utloggad.
      2. För användare med Windows 98 och Windows ME kan säkerhetskopiering utföras under utloggning efter design.

      När inställningarna för schemat är klara kan du gå ur Synology Data Replicator 3 eller minimera programfönstret till systemverktygsfältet. Synology Data Replicator 3 kommer att utföra den planerade säkerhetskopieringen vid den valda tidpunkten.

    • Ta bort schema

      • Klicka på Ta bort schema för att avbryta schemalagd säkerhetskopiering och ta bort schemainställningarna.

    Om du vill säkerhetskopiera filer från Outlook, Outlook Express eller Windows Mail observera att:

    • Om du kör Windows XP, Windows 2003 eller Windows Vista:
      1. måste minst en av diskenheterna på din dator vara i NTFS-format.
      2. Outlook-/Windows Mail-filer bör ligga på lokala diskenheter.
      3. Du måste vara inloggad som datorns administratör.
    • Om du kör andra operativsystem än Windows XP och Windows 2003, var vänlig stäng Outlook innan du kör säkerhetskopieringen.

      • Stöd & begränsningar

      1. För att söka efter en målserver använder Synology Data Replicator 3 portarna 9999, 9998 eller 9997.
      2. Komprimerade filer och krypterade filer kommer att bli vanliga filer efter säkerhetskopieringen. Filens övriga egenskaper kommer att raderas.
      3. Synology Data Replicator 3 stöder upp till 240 bytes långa filnamn.
      4. Om du ändrar namnet på din dator måste du ange administratörslösenordet för datorn när du schemalägger säkerhetskopiering, för att vara säker på att Data Replicator 3 utför den korrekt. Om du redan har ställt in ett säkerhetskopieringsschema måste du radera det schemat och använda administratörens lösenord för datorn för att ställa in ett nytt.

Återställning av säkerhetskopierade filer

För att återställa säkerhetskopierade filer måste du först välja en målmapp för säkerhetskopieringen och därefter välja en av följande metoder för att återställa dina säkerhetskopierade filer:

  1. Återställ via bläddring

    2. I de flesta situationer, när du vet var de säkerhetskopierade filerna ligger, kan du använda funktionen Återställ via bläddring för att återställa dina filer:

    1. Klicka på knappen Starta under Återställ via bläddring.
    2. Klicka på Nästa.
    3. Markera källan att återställa från och klicka på Nästa.
    4. Välj om du vill lista filer Efter version eller Efter återställningstillfälle.
    5. Markera de(n) mapp/-ar du vill återställa och klicka på Nästa.
    6. Välj plats till vilken de säkerhetskopierade filerna kommer att återställas. Klicka därefter på Slutför.

  2. Återställ via sökning

    3. Om du inte vet var de filer du vill återställa finns någonstans, eller om du bara vill återställa filer som innehåller vissa sökord, gör du följande för att återställa:

    1. Klicka på knappen Starta under Återställ via sökning.
    2. Klicka på Nästa.
    3. Markera källan att återställa från och klicka på Nästa.
    4. Skriv in det sökord du vill söka efter i textrutan vid "Sökord:" textfält.
    5. Välj plats för säkerhetskopieringen vid "Säkerhetskopieringar:" rullgardinsmeny.
    6. Klicka på Sök.
    7. Välj de filer och mappar du vill återställa i ramen till höger och klicka på Nästa.
    8. Välj plats till vilken de säkerhetskopierade filerna kommer att återställas. Klicka därefter på Slutför.

    Tips för återställning av Outlook, Outlook Express, och Windows Mail Files

    När du väljer sökväg till målet kan du välja mellan följande:

    1. Ursprunglig sökväg
      2.

      Genom att välja detta alternativ behöver du inte manuellt importera e-post och adressbok - datorns data ersätts av de återställda filerna.

    2. Ny sökväg
      3.

      Genom att välja detta alternativ kan du återställa datan till en specificerad sökväg och manuellt importera e-post och adressbok via Outlook/Outlook Express/Windows Mail.

Avancerade inställningar

I avsnittet Alternativ kan du göra avancerade inställningar för säkerhetskopiering enligt följande:

Filter

Använd den här funktionen för att göra precisa säkerhetskopieringar. Genom att utesluta filer med vissa prefix eller filändelser ur säkerhetskopieringslistan kan du göra säkerhetskopieringen snabbare. Följ dessa steg för att fortsätta:

  1. Klicka på Filter.
  2. Klicka på Lägg till till höger om alternativen "Filändelse" eller "Prefix".
  3. Ange specifika kriterier, till exempel ".exe" för filändelse eller "~" för prefix.
  4. Klicka på OK för att stänga dialogrutan och klicka sedan på OK igen för att spara inställningarna.

Aktivera e-postmeddelanden

Använd den här funktionen för att få detaljer om säkerhetskopieringen via e-post, efter det att Schemalagd säkerhetskopiering har utförts.

  1. Klicka på SMTP-inställning.
  2. Ange giltig e-postinformation och klicka på OK för att slutföra.

Aktivera filversioner

Vad är funktionen för filversioner till för?

Genom att aktivera funktionen för filversioner kan du spara en historik med olika versioner av samma fil.

Efter att den första säkerhetskopieringen har slutförts lagras alla de kopierade filerna i mappen "latest" vid målet för säkerhetskopian. Från och med nästa säkerhetskopiering kommer Data Replicator 3 automatiskt att kontrollera om någon fil med samma filnamn redan finns i mappen "latest". Om så är fallet, och innehållet i filen har ändrats, kommer den tidigare säkerhetskopierade filen i mappen "latest" att flyttas till mappen "past" som en äldre version, medan filen på din dator kommer att säkerhetskopieras till den mappen "latest". Där hittar du alltid de senast säkerhetskopierade filerna.

Obs!

Filen som hänvisas till ovan måste ligga i samma mapp. Så när du använder samma konto för att säkerhetskopiera filer genereras tidigare versioner.

Om du råkar ut för felaktig och oåterkallelig redigering eller förlust av dina lokala filer, och har aktiverat funktionen för filversioner, kan du på så sätt hitta tidigare filer i målmappen för säkerhetskopian och enkelt återställa dessa.

Antal tidigare versioner

Du kan ange hur många tidigare versioner av säkerhetskopierade filer du vill behålla. Maximalt antal versioner är 20. De tidigare versionernas filnamn får ett tillägg av "v00", "v01" och så vidare. Ju högre nummer på versionen desto nyare är den.

Ange ett antal för tidigare versioner, för att bestämma hur många versioner du vill spara vid målet för säkerhetskopian. Följ dessa steg för att fortsätta:

  1. Markera Aktivera filversioner.
  2. Ange en siffra för antal tidigare versioner och klicka på Tillämpa för att spara inställningarna.

Återställningstillfälle

Vad är återställningstillfällen?

Varje gång en säkerhetskopiering utförs, skapas en mapp som får namn efter säkerhetskopieringens starttid och innehåller alla filer som hör till denna säkerhetskopiering. Detta är ett återställningstillfälle.

Filer som aldrig har ändrats kommer att finnas med vid alla återställningstillfällen efter varje säkerhetskopiering. De tar dock bara upp diskutrymme motsvarande en fil.

Antal återställningstillfällen

Du kan ange hur många återställningstillfällen du vill behålla. Maximalt antal versioner är 128. Då antalet återställningstillfällen överskrider detta angivna värde kommer det äldsta tillfället i målmappen för säkerhetskopian att automatiskt raderas vid nästa säkerhetskopiering.

Stöd & begränsningar
Återställningstillfälle-funktionen stöds endast under Windows 2000 / Windows XP / Windows 2003 / Windows Vista.

Automatisk start vid inloggning

Du kan göra en inställning så att Synology Data Replicator 3 startar automatiskt när du loggar in i Windows. Följ dessa steg för att fortsätta:

  1. Markera Auto-starta Data Replicator 3 när Windows startas.
  2. Klicka på Tillämpa.

Kontrollera dataöverensstämmelse före synkronisering

Du kan välja att kontrollera dataöverensstämmelsen innan Data Replicator går in i synkläge. Följ dessa steg för att fortsätta:

  1. Markera Kontrollera dataöverensstämmelse före synkronisering.
  2. Klicka på Tillämpa.

Obs!

Om du avmarkerar det här alternativet och stänger Data Replicator finns det risk för att data som lagts till, tagits bort eller ändrats inte synkroniseras till servern när du startar Data Replicator nästa gång.

Behåll identiska filer på målservern.

Du kan ange att de filer i mappen "latest" som är identiska med dem på din lokala dator skall behållas. Följ dessa steg för att fortsätta:

  1. Markera När lokala filer tas bort, radera motsvarande säkerhetskopierade filer på samma gång.
  2. Klicka på Tillämpa.

Loggfil

Maximal storlek för loggfilen är 512kB. När loggfilen når denna gräns kommer de äldsta 128kB att tas bort.

  • Spara loggfil
    1. Klicka på Spara.
    2. Välj sökväg till målet för säkerhetskopian och klicka på Spara för att avsluta.

  • Rensa loggfil
    1. Klicka på Rensa.
    2. Klicka på Ja i dialogrutan för att bekräfta.

備份PC 檔案

請按照下列順序設定備份。在開始備份前，您亦可前往進階設定頁面進行進階備份設

  1. 選擇備份目的地

    2. 您可以選擇Synology 伺服器上的共用資料夾或是其他位置的資料夾作為備份目的地，未來，在備份成功執行後，您便可以在您所指定的目的地資料夾中找到備份檔案。

    1.1 備份到Synology 伺服器

    1. 點選選擇按鈕。
    2. 選擇Synology 伺服器，然後按確定。
    3. 在下一個畫面中點選共用資料夾所在的Synology 伺服器，然後按下一步。
    4. 輸入您存取該共用資料夾的帳號和密碼，再按下一步。
    5. 點選您要使用的共用資料夾，然後按完成。

    1.2 備份到其他位置

    1. 點選選擇按鈕。
    2. 選擇其他位置，然後按瀏覽選擇備份目的地。
    3. 按確定完成。

  2. 勾選想要備份的資料夾

  3. 選擇備份模式

    4. Data Replicator 3 提供3 種備份方式：立即備份、同步、排程備份。

    3.1 立即備份

         按下立即備份，將勾選的資料夾內所有檔案立即備份到備份目的地。

    3.2 同步

    • 啟動同步

      • 按下同步，Data Replicator 3 會將勾選的資料夾內所有檔案立即備份到備份目的地，然後進入監控狀態，保持資料同步。若資料夾中的檔案有任何修改，則會被備份到備份目的地。

      注意：

      1. 當您啟動同步後，同步功能會持續進行，直到您停止同步。
      2. 同步電子郵件的時間間隔指的是從同步完成後至下次同步開始之間的時間。實際執行同步電子郵件的時間間隔可能會因為Data Replicator 3 正在同步其他檔案而延長。

    • 停止同步

      • 請點選程式視窗中的停止同步。若您的程式視窗已經最小化至系統列，請雙擊Data Replicator 3 程式圖示開始程式視窗。

    3.3 排程備份

    • 啟動排程備份

      • 您可以設定Data Replicator 3 固定在指定的時間自動進行備份。請依照下列步驟設定：

      1. 按下排程備份。
      2. 選擇排程備份的週期和開始的時間，然後按確定。
      3. 若您希望排程備份在您登出電腦時依然正常執行，請輸入您登入本電腦的使用者帳號和密碼，然後按確定。
        若您不需要在登出時執行排程備份，請按略過。

      注意：

      1. 密碼欄位不可為空白。
      2. 在Windows 98 或Windows ME 環境下，使用者無需登入即可進行排程備份。

      設定完畢後，您可以關閉Data Replicator 3 或是將程式縮小。每當指定排程備份的時間一到，Data Replicator 3 會自動進行排程備份。

    • 刪除排程

      • 請點選程式視窗中的刪除排程。

    若您想要備份Outlook、Outlook Express 及Windows Mail 檔案，請注意下列幾點：

    • 若您的作業環境為Windows XP、Windows 2003，或Windows Vista：
      1. 您電腦上使用的硬碟機必須至少有一個是格式化為NTFS 格式。
      2. Outlook/Windows Mail 檔案必須存在於本機硬碟中。
      3. 您必須以這台電腦的系統管理員身分登入。
    • 若您的作業環境為Windows XP 或Windows 2003 以外的作業系統，則請先關閉Outlook 再進行備份。

      • 支援與限制

      1. 在尋找目標伺服器的時候，Synology Data Replicator 3 會依序使用連接埠9999、或9998、或是9997。
      2. 壓縮和加密這兩種檔案屬性在備份後會消失，即具此屬性的檔案在備份後會變為一般檔案。
      3. Synology Data Replicator 3 支援至多240 個位元數的長檔名檔案備份。
      4. 若您修改了您電腦的名稱，則在設定排程備份時必須輸入您電腦的管理員密碼，以確保Data Replicator 3 正常執行。若您在電腦上已設定執行排程備份，則需刪除排程，重新以輸入電腦管理員密碼的方式設定一次。

回復備份檔案

在選擇備份目的地後，您可以使用下列2 種檔案回復方式回復您需要的檔案：

  1. 以瀏覽的方式來回復

    2. 一般狀況下，如果您確定要回復的檔案位置，請依照下列步驟用瀏覽的方式來回復：

    1. 按下「以瀏覽的方式來回復」項目下的開始。
    2. 按下一步。
    3. 選擇回復檔案來源，按下一步。
    4. 選擇您要依照版本或回復點來尋找資料。
    5. 要回復的資料夾及檔案，按下一步。
    6. 選擇目標路徑，然後按完成。

  2. 以搜尋的方式來回復

    3. 如果您不確定要回復的檔案位置，或只想回復特定檔名的檔案，請依照下列步驟用搜尋的方式來回復：

    1. 按下「以搜尋的方式來回復」項目下的開始。
    2. 按下一步。
    3. 選擇回復檔案來源，按下一步。
    4. 在「關鍵字：」文字方塊下輸入要搜尋檔案的關鍵字。
    5. 在「備份：」下拉選單中選擇備份檔案。
    6. 按下搜尋。
    7. 勾選右方視窗中要回復的資料夾及檔案，然後按下一步。
    8. 選擇目標路徑，然後按完成。

    回復Outlook、Outlook Express 及Windows Mail 檔案小技巧

         在選擇目標路徑時，您可以選擇將檔案回復到：

    1. 原始路徑
      2.

      選擇原始路徑，您可省去手動匯入郵件或通訊錄的動作，不過您PC端的資料會被回復的檔案取代。

    2. 新路徑
      3.

      選擇新路徑，您可以直接將檔案回復到指定的位置，但回復後請使用Outlook/Outlook Express/Windows Mail的匯入功能匯入郵件及通訊錄。

進階設定

在設定頁面，您可以為您的PC 備份做進階的設定。

檔案過濾

使用檔案過濾可以幫助您精準地備份檔案，排除檔案名稱包含特定字首或副檔名的檔案以縮短備份的時間。請依照下列步驟設定：

  1. 點選檔案過濾按鈕。
  2. 按「副檔名」或「字首」欄位右方的新增。
  3. 輸入條件或副檔名，例如：副檔名為「.exe」或字首為「~」的檔案。
  4. 按確定，並且按套用儲存設定。

啟動電子郵件通知

啟動電子郵件通知選項，您可以在排程備份完成後收到電子郵件通知該次備份的詳細資訊。

  1. 點選SMTP 設定按鈕。
  2. 輸入有效的電子郵件資訊，並且按OK 完成。

啟用檔案版本功能

什麼是檔案版本功能？

檔案版本功能讓您保留同一檔案的歷史修改記錄。

第一次成功備份後，所有檔案將被存放在「latest」資料夾。之後每一次執行備份時，程式皆自動檢查「latest」資料夾是否已存有相同檔名的檔案。若已存有相同檔名的檔案但內容相異（表示該檔案曾被修改），則前次備份的該檔案將被視為舊版本，並自動轉存至「past」資料夾；現行檔案則被視為最新版本，並自動存放於「latest」資料夾。 因此，您可以在「latest」資料夾找到所有備份檔案的最新版本。

注意：

該檔案需位於同一個共用資料夾，且由同一個使用者進行備份，才被視為同一檔案而產生新舊版本。

若您啟用了檔案版本功能，當本機資料遭到不當修改且無法回復時，您將可以從伺服器端的備份資料庫中找到歷史檔案，並藉由回復功能回復該檔案。

舊版本數目

您可以自行決定欲保留的舊版本數目，至多20個版本。舊版本檔案的檔名後方將會自動加上數字標註（v00、v01等以此類推），數字越大表示該檔案越新。

利用此功能設定要在備份目的地保留幾個舊檔案備份版本，請依照下列步驟設定：

  1. 勾選啟用檔案版本功能。
  2. 設定舊版本數目，然後按套用儲存設定。

回復點

什麼是回復點？

在每一次成功備份勾選的檔案後，備份目的地即會產生一個以備份開始時間為檔名的資料夾，這個資料夾就是一個回復點，包含此次備份的所有檔案。

同一檔案若未經修改而再次備份，仍會存在於每個回復點中，但在硬碟中只保留一份實體檔案，並不重複佔用硬碟空間。

回復點個數

您可以自行決定欲保留的回復點個數，至多128個。若回復點數目已超出所設定的數值時，則伺服器上較舊的回復點會在下次備份完成後自動被刪除。

支援與限制
回復點功能只支援於Windows 2000 / Windows XP / Windows 2003 / Windows Vista。

登入時自動啟動

您可以設定在登入 Windows 後，自動啟動 Synology Data Replicator 3。請依照下列步驟設定：

  1. 勾選當我登入 Windows 時自動啟動 Data Replicator 3。
  2. 按套用。

同步前檢查檔案一致性

您可以決定是否要在 Data Replicator 進入同步模式之前檢查檔案一致性。請依照下列步驟設定：

  1. 請勾選同步前檢查檔案一致性。
  2. 按套用。

注意：

如果您取消勾選此選項並關閉 Data Replicator，然後開始新增、刪除或修改資料，則當您下次啟動 Data Replicator 時，這些資料可能不會同步至伺服器上。

保持備份目的地端檔案一致

您可以設定讓備份目的地上「latest」資料夾中的備份檔案與PC 端備份來源目錄中的檔案保持一致，相互對應。請依照下列步驟設定：

  1. 勾選當刪除本機電腦上的檔案時，同時刪除相對應的備份檔案。
  2. 按套用。

日誌紀錄

日誌的容量最大是 512KB，達到容量上限時，系統會將較舊的128KB 記錄刪除。

  • 儲存日誌
    1. 按程式視窗下方的儲存。
    2. 選取儲存位置，然後按儲存完成。

  • 清除日誌
    1. 按程式視窗下方的清除。
    2. 確認清除日誌，請按確認視窗的是。

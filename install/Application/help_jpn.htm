Data Replicator 3 を使用する

次の良く使う機能をクリックしてData Replicator 3の使い方を学んでください。

  • PCデータバックアップ
  • バックアップファイルの復元
  • 詳細設定
  • ログ

PCデータバックアップ

次の手順に従ってバックアップを設定してください。PCデータのバックアップをカスタマイズするさらに詳しい設定については詳細設定をご覧ください。

  1. バックアップ先の選択

    2. Synology サーバーまたは他の場所にバックアップするPCデータを選択します。バックアップが完了すると、指定したフォルダ内にバックアップファイルが保存されます。

    1.1 Synology サーバーへのバックアップ

    1. 選択をクリックします。
    2. Synology サーバーを選択し、OK をクリックします。
    3. 次のウィンドウで、バックアップ先の共有フォルダのあるSynologyサーバーをクリックします。次に、次へをクリックします。
    4. 選択した共有フォルダを読込み/書込みできるユーザー名とパスワードを入力してから次へをクリックします。
    5. バックアップに使用したい共有フォルダをクリックしてから終了をクリックします。

    1.2 その他の場所へのバックアップ

    1. 選択をクリックします。
    2. 「その他の場所」を選んでから「参照」をクリックしてバックアップ先を指定します。
    3. OK をクリックして終了します。

  2. バックアップするフォルダの選択

  3. バックアップモードの選択

    4. Data Replicator 3 はとても便利な3種類モードを提供しています。3つのモードとは、即時、同期、予約です。

    3.1 今すぐ実行

    「即時」ボタンをクリックすると、選択したローカルフォルダ内のすべてのファイルを即時にバックアップ先にバックアップします。

    3.2 同期

    • 即座にデータを同期して監視を開始する

      • 「同期」ボタンをクリックすると、Data Replicator 3 は選択したフォルダのファイルをすべてバックアップ先に完全バックアップします。バックアップが終了すると、Data Replicator 3 はそのフォルダの監視を開始します。変更があると、Data Replicator 3 は変更したファイルのみバックアップします。

      注意：

      1. Data Replicator 3 が監視モードに入ると停止するまで監視し続けます。
      2. メールの同期化実施周期は、同期化が完了した時間から次の同期化が始まるまでの時間です。この処理は Data Replicator 3 が他のファイルの同期化処理を行っている場合、延期されます。

    • 監視を停止する

      • 監視を停止するには、Data Replicator 3 プログラムウィンドウの「同期停止」ボタンをクリックします。プログラムウィンドウがシステムトレイに最小化されている場合は、Data Replicator 3 アイコンを ダブルクリックすると開きます。

    3.3 スケジュール

    • 予約の設定

      • Data Replicator 3 は指定した時間に定期的にバックアップする自動バックアップを設定することもできます。以下の手順で行います。

      1. 予約をクリックします。
      2. スケジュール バックアップするためのバックアップ期間と開始時間を設定してから OK をクリックします。
      3. ログオフ時にスケジュール バックアップを実施するには、コンピュータを使用するために使うユーザー名とパスワードを入力してください。次に OK をクリックします。
        ログオフ時にスケジュール バックアップを実施しない場合は、スキップをクリックします。

      注意：

      1. ログアウト中にバックアップを実施するには、ユーザーパスワード欄を空白にすることはできません。
      2. Windows 98 または Windows MEをお使いの方は、ログアウトしていてもスケジュール バックアップを実施できるように設計されています。

      予約の設定が完了したら、Synology Data Replicator 3を終了するか、システムトレイにプログラムウィンドウを最小化します。Synology Data Replicator 3 は時間がきたら自動的にスケジュール・バックアップを実行します。

    • 予約の削除

      • 予約の削除をクリックするとスケジュール バックアップを停止して予約の設定を削除します。

    Outlook、Outlook Express、またはWindows メールファイルをバックアップする場合には、以下の点に留意してください。

    • Windows XP、Windows 2003、またはWindows Vistaを起動している場合、
      1. PCの1つ以上のドライブがNTFSでフォーマットされている必要があります。
      2. Outlook/Windows メールファイルはローカルのドライブにある必要があります。
      3. PCの管理者としてログインする必要があります。
    • Windows XPおよびWindows 2003以外のオペレーティングシステムを起動している場合は、Outlookを閉じてからOutlookのバックアップを行ってください。

      • サポートと制限

      1. ターゲットサーバーを検索するとき、Synology Data Replicator 3 はポート9999、9998、または9997を使用します。
      2. 圧縮ファイルと暗号化ファイルは、バックアップ後に通常のファイルになります。ファイルのプロパティは消去されます。
      3. Synology Data Replicator 3 では、最大長240バイトのファイル名に対応しています。
      4. PCのコンピュータ名を変更する場合は、PCの管理者ログインパスワードを入力してからスケジュールバックアップを設定してData Replicator 3が正しく動作するようにします。バックアップの予約が既に設定されている場合には、その予約を削除してPCの管理者ログインパスワードを使用してもう一度新しく設定します。

バックアップファイルの復元

バックアップファイルを復元するには、バックアップターゲットを先に選択する必要があります。バックアップファイルの復元には次の方法のいずれかを選びます。

  1. 閲覧して復元

    2. 多くの場合、バックアップファイルの場所が分かっているときは閲覧して復元を使ってファイルを復元できます。

    1. 閲覧して復元の開始ボタンをクリックします。
    2. 次へ をクリックします。
    3. 復元元を選択してから次へをクリックします。
    4. バージョンまたは復元ポイントのどちらでファイルを閲覧するか選択します。
    5. 復元するフォルダを選んでから次へをクリックします。
    6. バックアップファイルを復元する場所を選択します。終了をクリックします。

  2. 検索して復元

    3. バックアップファイルの場所がわからない、または特定のキーワードを使って復元したい場合は次の手順で行います。

    1. 検索して復元の開始ボタンをクリックします。
    2. 次へ をクリックします。
    3. 復元元を選択してから次へをクリックします。
    4. [キーワード:]テキストフィールドに、検索するキーワードを入力します。
    5. [バックアップ:]ドロップダウンメニューから、からバックアップ時間を選択します。
    6. 検索をクリックします。
    7. 右側の枠に復元したいファイルとフォルダを選択してから次へをクリックします。
    8. バックアップファイルを復元する場所を選択します。終了をクリックします。

    Outlook、Outlook Express、およびWindowsメールファイルを復元する場合のヒント

    バックアップ先パスを選択する場合、次のいずれかを選びます。

    1. オリジナルパス
      2.

      このオプションを選択すると、電子メールやアドレス帳を手動でインポートする必要はありません。PCは復元されたファイルで置き換えられます。

    2. 新しいパス

      3. このオプションを選択すると、指定したパスにデータを保存することができます。Outlook/Outlook Express/Windows メールの電子メールやアドレス帳は手動でインポートしてください。

詳細設定

オプション欄では、バックアップ用に次の詳細が設定できます。

フィルタ

この機能でバックアップが正確に行えます。特定の接頭辞または拡張子を持つファイルをバックアップリストから除外すると、バックアップの時間短縮になります。次のステップに従って続行してください。

  1. [フィルタ] をクリックします。
  2. 「接頭辞」または「拡張子」オプションの右側にある追加をクリックします。
  3. 例えば、拡張子の「exe」、または接頭辞の「~」などの指定条件を入力します。
  4. OK をクリックしてセットアップダイアログを閉じ、再度 OK をクリックして設定を保存します。

電子メール通知の実施

この機能を使うと、スケジュール バックアップを実施した後にバックアップの詳細を電子メールで受け取ります。

  1. SMTP設定をクリックします。
  2. 有効な電子メールの情報を入力してから OK をクリックして終了します。

ファイルバージョンの実施

ファイルバージョン機能とは？

ファイルバージョン機能を実行すると同じファイルのバージョン履歴を保存することができます。

最初のバックアップが完了すると、すべてのバックアップファイルがバックアップ先の「latest」フォルダに保存されます。次回のバックアップから、Data Replicator 3 は同じファイル名が「latest」フォルダに存在するかを自動的に確認します。同じファイルが存在し、それが変更されている場合は、「latest」フォルダにある前回バックアップしたファイルは古いファイルとして「past」フォルダに移され、コンピュータにある新しいファイルが「latest」フォルダにバックアップされます。このようにして「latest」フォルダには常に最新のファイルが保存されます。

注意：

参照されるファイルは必ず同じフォルダに存在している必要があります。同じアカウントでフォルダをバックアップをする場合、常に古いバージョンのファイルが生成されます。

ファイルバージョン機能を実行中にローカルファイルが不適切に編集されたり取り消し不可状態になった場合、バックアップ先の履歴ファイルを使って簡単に復元することができます。

古いバージョンの数

保存したいバックアップファイルの古いバージョン数をここで指定できます。最大は20です。古いバージョンファイルのファイル名は「v00」、「v01」のように付けられます。バージョン番号が大きいほど新しいバックアップであることを表します。

古いバージョンの数を設定してバックアップ先に保管するバージョンの数を決めます。次のステップに従って続行してください。

  1. ファイルバージョンの実行をクリックします。
  2. 古いバージョンの数を設定してから適用をクリックして設定を保存する。

復元ポイント

復元ポイントとは？

バックアップを実施すると毎回バックアップ開始時間を名前にしたフォルダが生成されます。このフォルダにはバックアップのすべてのファイルが保存されています。これが復元ポイントです。

全く変更のないファイルはそれぞれのバックアップが終了した後の復元ポイントに毎回記載されます。ただし、実際はファイルディスク1枚分のサイズしか使用していません。

復元ポイントの数

ここで保管する復元ポイントの数を設定できます。最大は128です。復元ポイント数が設定値を超えた場合は、次のバックアップを行った後、バックアップ先の最も古い復元ポイントから自動的に削除されます。

サポートと制限
復元ポイント機能は、Windows 2000 / Windows XP / Windows 2003 / Windows Vistaでのみ使用できます。

ログオン時の自動起動

Windows にログインしたときに自動的に Synology Data Replicator 3 を起動するように設定することができます。次のステップに従って続行してください。

  1. Windows起動時にData Replicator 3 を自動起動をクリックします。
  2. 適用をクリックします。

同期する前にデータの整合性を確認する

Data Replicator が同期モードに入る前に、データの整合性をチェックするかどううかを決めることができます。次のステップに従って続行してください。

  1. 同期する前にデータの整合性を確認するを選択します。
  2. 適用をクリックします。

注意：

このオプションを選択せずに Data Replicator を閉じると、追加、削除、または変更されたデータは、Data Replicator を次に起動したときにサーバーと同期されない場合があります。

サーバーに同じファイルを保存

ローカルコンピュータのファイルと同じファイルを「latest」フォルダに保存するよう設定することができます。次のステップに従って続行してください。

  1. ローカルファイルが削除されると同時に反対のバックアップファイルも削除するを選択します。
  2. 適用をクリックします。

ログ

最大ログファイルは 512KB です。ログファイルがそのサイズに達すると、最も古いものから 128KB のデータが削除されます。

  • ログを保存
    1. 保存をクリックします。
    2. バックアップ先のパスを選択してから「保存」をクリックして終了します。

  • ログを削除
    1. 消去をクリックします。
    2. 確認ダイアログのはいをクリックします。

备份PC 文件

请按照下列顺序设置备份。在开始备份前，您亦可前往高级设定页面进行进阶备份设

  1. 选择备份目的地

    2. 您可以选择Synology 服务器上的共享文件夹或是其他位置的文件夹作为备份目的地，未来，在备份成功运行后，您便可以在您所指定的目的地文件夹中找到备份文件。

    1.1 备份到Synology 服务器

    1. 点选选择按钮。
    2. 选择Synology 服务器，然后按确定。
    3. 在下一个画面中点选共享文件夹所在的Synology 服务器，然后按下一步。
    4. 输入您存取该共享文件夹的账号和密码，再按下一步。
    5. 点选您要使用的共享文件夹，然后按完成。

    1.2 备份到其他位置

    1. 点选选择按钮。
    2. 选择其他位置，然后按浏览选择备份目的地。
    3. 按确定完成。

  2. 勾选想要备份的文件夹

  3. 选择备份模式

    4. Data Replicator 3 提供3 种备份方式：立即备份、同步、排程备份。

    3.1 立即备份

         按下立即备份，将勾选的文件夹内所有文件立即备份到备份目的地。

    3.2 同步

    • 启动同步

      • 按下同步，Data Replicator 3 会将勾选的文件夹内所有文件立即备份到备份目的地，然后进入监控状态，保持数据同步。若文件夹中的文件有任何修改，则会被备份到备份目的地。

      注意：

      1. 当您启动同步后，同步功能会持续进行，直到您停止同步。
      2. 同步电子邮件的时间间隔指的是从同步完成后至下次同步开始之间的时间。实际运行同步电子邮件的时间间隔可能会因为Data Replicator 3 正在同步其他文件而延长。

    • 停止同步

      • 请点选程序窗口中的停止同步。若您的程序窗口已经最小化至系统列，请双击Data Replicator 3 程序图标开始程序窗口。

    3.3 排程备份

    • 启动排程备份

      • 您可以设置Data Replicator 3 固定在指定的时间自动进行备份。请依照下列步骤设置：

      1. 按下排程备份。
      2. 选择排程备份的周期和开始的时间，然后按确定。
      3. 若您希望排程备份在您注销计算机时依然正常运行，请输入您登入本计算机的用户账号和密码，然后按确定。
        若您不需要在注销时运行排程备份，请按略过。

      注意：

      1. 密码字段不可为空白。
      2. 在Windows 98 或Windows ME 环境下，使用者无需登入即可进行排程备份。

      设置完毕后，您可以关闭Data Replicator 3 或是将程序缩小。每当指定排程备份的时间一到，Data Replicator 3 会自动进行排程备份。

    • 删除排程

      • 请点选程序窗口中的删除排程。

    若您想要备份Outlook、Outlook Express 及Windows Mail 文件，请注意下列几点：

    • 若您的工作环境为Windows XP、Windows 2003，或Windows Vista：
      1. 您计算机上使用的硬盘驱动器必须至少有一个是格式化为NTFS 格式。
      2. Outlook/Windows Mail 文件必须存在于本机硬盘中。
      3. 您必须以这台计算机的系统管理员身分登入。
    • 若您的工作环境为Windows XP 或Windows 2003 以外的操作系统，则请先关闭Outlook 再进行备份。

      • 支持与限制

      1. 在寻找目标服务器的时候，Synology Data Replicator 3 会依序使用端口9999、或9998、或是9997。
      2. 压缩和加密这两种文件属性在备份后会消失，即具此属性的文件在备份后会变为一般文件。
      3. Synology Data Replicator 3 支持至多240 个比特数的长文件名文件备份。
      4. 若您修改了您计算机的名称，则在设置排程备份时必须输入您计算机的管理员密码，以确保Data Replicator 3 正常运行。若您在计算机上已设置运行排程备份，则需删除排程，重新以输入计算机管理员密码的方式设置一次。

回复备份文件

在选择备份目的地后，您可以使用下列2 种文件回复方式回复您需要的文件：

  1. 以浏览的方式来回复

    2. 一般状况下，如果您确定要回复的文件位置，请依照下列步骤用浏览的方式来回复：

    1. 按下「以浏览的方式来回复」项目下的开始。
    2. 按下一步。
    3. 选择回复文件来源，按下一步。
    4. 选择您要依照版本或回复点来寻找数据。
    5. 要回复的文件夹及文件，按下一步。
    6. 选择目标路径，然后按完成。

  2. 以搜索的方式来回复

    3. 如果您不确定要回复的文件位置，或只想回复特定文件名的文件，请依照下列步骤用搜索的方式来回复：

    1. 按下「以搜索的方式来回复」项目下的开始。
    2. 按下一步。
    3. 选择回复文件来源，按下一步。
    4. 在「关键词：」文本框下输入要搜索文件的关键词。
    5. 在「备份：」下拉选单中选择备份文件。
    6. 按下搜索。
    7. 勾选右方窗口中要回复的文件夹及文件，然后按下一步。
    8. 选择目标路径，然后按完成。

    回复Outlook、Outlook Express 及Windows Mail 文件小技巧

         在选择目标路径时，您可以选择将文件回复到：

    1. 原始路径
      2.

      选择原始路径，您可省去手动汇入邮件或通讯簿的动作，不过您PC端的数据会被回复的文件取代。

    2. 新路径
      3.

      选择新路径，您可以直接将文件回复到指定的位置，但回复后请使用Outlook/Outlook Express/Windows Mail的汇入功能汇入邮件及通讯簿。

高级设定

在设置页面，您可以为您的PC 备份做进阶的设置。

文件过滤

使用文件过滤可以帮助您精准地备份文件，排除文件名包含特定前缀或扩展名的文件以缩短备份的时间。请依照下列步骤设置：

  1. 点选文件过滤按钮。
  2. 按「扩展名」或「前缀」字段右方的新增。
  3. 输入条件或扩展名，例如：扩展名为「.exe」或前缀为「~」的文件。
  4. 按确定，并且按套用存储设置。

启动电子邮件通知

启动电子邮件通知选项，您可以在排程备份完成后收到电子邮件通知该次备份的详细信息。

  1. 点选SMTP 设置按钮。
  2. 输入有效的电子邮件信息，并且按OK 完成。

启用文件版本功能

什么是文件版本功能？

文件版本功能让您保留同一文件的历史修改记录。

第一次成功备份后，所有文件将被存放在「latest」文件夹。之后每一次运行备份时，程序皆自动检查「latest」文件夹是否已存有相同文件名的文件。若已存有相同文件名的文件但内容相异（表示该文件曾被修改），则前次备份的该文件将被视为旧版本，并自动转存至「past」文件夹；现行文件则被视为最新版本，并自动存放于「latest」文件夹。 因此，您可以在「latest」文件夹找到所有备份文件的最新版本。

注意：

该文件需位于同一个共享文件夹，且由同一个使用者进行备份，才被视为同一文件而产生新旧版本。

若您启用了文件版本功能，当本机数据遭到不当修改且无法回复时，您将可以从服务器端的备份数据库中找到历史文件，并藉由回复功能回复该文件。

旧版本数目

您可以自行决定欲保留的旧版本数目，至多20个版本。旧版本文件的文件名后方将会自动加上数字标注（v00、v01等以此类推），数字越大表示该文件越新。

利用此功能设置要在备份目的地保留几个旧文件备份版本，请依照下列步骤设置：

  1. 勾选启用文件版本功能。
  2. 设置旧版本数目，然后按套用存储设置。

回复点

什么是回复点？

在每一次成功备份勾选的文件后，备份目的地即会产生一个以备份开始时间为文件名的文件夹，这个文件夹就是一个回复点，包含此次备份的所有文件。

同一文件若未经修改而再次备份，仍会存在于每个回复点中，但在硬盘中只保留一份实体文件，并不重复占用硬盘空间。

回复点个数

您可以自行决定欲保留的回复点个数，至多128个。若回复点数目已超出所设置的数值时，则服务器上较旧的回复点会在下次备份完成后自动被删除。

支持与限制
回复点功能只支持于Windows 2000 / Windows XP / Windows 2003 / Windows Vista。

登入时自动启动

您可以设置在登入 Windows 后，自动启动 Synology Data Replicator 3。请依照下列步骤设置：

  1. 勾选当我登入 Windows 时自动启动 Data Replicator 3。
  2. 按套用。

同步前检查文件一致性

您可以决定是否要在 Data Replicator 进入同步模式之前检查文件一致性。请依照下列步骤设置：

  1. 请勾选同步前检查文件一致性。
  2. 按套用。

注意：

如果您取消勾选此选项并关闭 Data Replicator，然后开始新增、删除或修改数据，则当您下次启动 Data Replicator 时，这些数据可能不会同步至服务器上。

保持备份目的地端文件一致

您可以设置让备份目的地上「latest」文件夹中的备份文件与PC 端备份来源目录中的文件保持一致，相互对应。请依照下列步骤设置：

  1. 勾选当删除本机计算机上的文件时，同时删除相对应的备份文件。
  2. 按套用。

日志纪录

日志的容量最大是 512KB，达到容量上限时，系统会将较旧的128KB 记录删除。

  • 存储日志
    1. 按程序窗口下方的存储。
    2. 选取存储位置，然后按存储完成。

  • 清除日志
    1. 按程序窗口下方的清除。
    2. 确认清除日志，请按确认窗口的是。

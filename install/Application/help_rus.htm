Использование Data Replicator 3

Чтобы ознакомиться с программой Data Replicator 3, нажимайте следующие основные функции.

  • Резервирование данных ПК
  • Восстановление резервных копий файлов
  • Дополнительные настройки
  • Журналы

Резервирование данных ПК

Для настройки резервирования выполните следующие действия. Можно также перейти в раздел Дополнительные настройки, чтобы узнать о других настройках, которые помогут вам индивидуализировать резервирование данных ПК.

  1. Выберите место для резервирования

    2. Можно выполнить резервирование данных ПК на сервер Synology или в другое место. После успешного завершения резервирования вы сможете найти файлы резервирования в заданной папке.

    1.1 Резервирование на сервер Synology

    1. Нажмите кнопку Выбрать.
    2. Выберите Сервер Synology и нажмите кнопку OK.
    3. В следующем окне нажмите сервер Synology, на котором расположена папка общего доступа для резервирования. Затем нажмите Далее.
    4. Введите имя пользователя и пароль с правом чтения/записи выбранной папки общего доступа, затем нажмите Далее.
    5. Нажмите папку общего доступа, которая будет использоваться для резервирования, затем нажмите Готово.

    1.2 Резервирование в другое место

    1. Нажмите кнопку Выбрать.
    2. Выберите Другое место и нажмите кнопку Обзор, чтобы задать путь резервирования.
    3. Нажмите кнопку OK, чтобы завершить.

  2. Установите флажок в папки для резервирования

  3. Выберите режим резервирования

    4. Data Replicator 3 обеспечивает 3 режима резервирования для максимального удобства, а именно: Немедленное, Синхронизация и Запланированное.

    3.1 Немедленное

         При нажатии на кнопку Немедленное, все файлы в выбранной локальной папке(ах) будут немедленно скопированы в место, выбранное для резервирования.

    3.2 Синхронизовать

    • Синхронизовать данные сейчас и начать мониторинг

      • При нажатии на кнопку Синхронизация, Data Replicator 3 выполнит полное резервирование всех файлов в выбранной папке(ах) в место, выбранное для резервирования. После этого Data Replicator 3 начнет мониторинг этой папки (папок). По мере внесения изменений Data Replicator 3 выполняет резервирование только тех файлов, которые подверглись изменениям.

      Примечание.

      1. После того, как Data Replicator 3 перейдет в режим мониторинга, программа продолжит работать до тех пор, пока вы ее не остановите.
      2. Интервал времени между синхронизацией почты означает период времени между завершением одного сеанса синхронизации до начала следующего сеанса синхронизации. Задача может быть отложена, если Data Replicator 3 выполняет синхронизацию других файлов.

    • Остановить мониторинг

      • Чтобы остановить мониторинг, нажмите кнопку Остановить синхронизацию, в окне программы Data Replicator 3. Если окно программы было свернуто в панель задач, дважды нажмите значок Data Replicator 3, чтобы открыть программу.

    3.3 График

    • Настроить план

      • Программу Data Replicator 3 можно настроить на регулярное автоматическое резервирование через заданные промежутки времени. Для этого:

      1. Нажмите План.
      2. Задайте период резервирования и время начала Запланированного резервирования, затем нажмите OK.
      3. Если необходимо выполнить запланированное резервирование в отключенном состоянии, введите имя пользователя и пароль, используемые для входа в данный компьютер. Затем нажмите кнопку OK.
        Если запланированное резервирование не выполняется в отключенном состоянии, нажмите Пропустить.

      Примечание.

      1. Чтобы выполнить резервирование, не входя в систему, пароль пользователя может быть пустым.
      2. Пользователи Windows 98 или Windows ME могут выполнять запланированное резервирование, не входя в систему, благодаря конструкции операционной системы.

      По завершении настройки плана можно выйти из программы Synology Data Replicator 3 или свернуть окно программы в панель задач. Synology Data Replicator 3 будет автоматически выполнять запланированное резервирование в заданное время.

    • Удалить план

      • Чтобы остановить запланированное резервирование и удалить настройки плана, нажмите кнопку Удалить план.

    При резервировании файлов Outlook, Outlook Express или Windows Mail, учтите:

    • Если вы используете ОС Windows XP, Windows 2003 или Windows Vista,
      1. Хотя бы один из дисков на ПК должен быть в формате NTFS.
      2. Файлы Outlook/Windows Mail должны быть расположены на локальных дисках.
      3. Необходимо войти в систему под учетной записью администратора ПК.
    • Если используется операционная система, отличная от Windows XP и Windows 2003, закройте Outlook, прежде чем выполнять резервирование Outlook.

      • Поддержка и ограничение

      1. Во время поиска целевого сервера программа Synology Data Replicator 3 использует порт 9999, 9998 или 9997.
      2. Сжатые и зашифрованные файлы после резервирования становятся обычными файлами. Свойства файла будут стерты.
      3. Что касается длины имени файла, Synology Data Replicator 3 поддерживает до 240 байт.
      4. При изменении имени компьютера необходимо ввести пароль администратора ПК во время настройки запланированного резервирования, для корректной работы программы Data Replicator 3. Если план резервирования уже был настроен, удалите данный план и с помощью пароля администратора ПК создайте новый.

Восстановление резервных копий файлов

Чтобы восстановить файлы резервирования, сначала необходимо выбрать целевой файл резервирования, затем выбрать один из следующих методов восстановления файлов резервирования:

  1. Просмотр и восстановление

    2. В большинстве ситуаций, если вы точно знаете, где находятся файлы резервирования, для восстановления файлов можно использовать функцию Просмотр и восстановление:

    1. Нажмите кнопку Пуск, расположенную под кнопкой Просмотр и восстановление.
    2. Нажмите Далее.
    3. Нажмите, чтобы выбрать источник восстановления, затем нажмите Далее.
    4. Выберите метод сортировки файлов По версии или По точке восстановления.
    5. Установите флажок в папку(и), которые необходимо восстановить, затем нажмите Далее.
    6. Выберите путь восстановления файлов резервирования. Затем нажмите Готово.

  2. Поиск и восстановление

    3. Если вы не знаете точно, где находятся файлы резервирования, или хотите восстановить файлы с определенным ключевым словом, выполните следующие действия:

    1. Нажмите кнопку Пуск, расположенную под кнопкой Поиск и восстановление.
    2. Нажмите Далее.
    3. Нажмите, чтобы выбрать источник восстановления, затем нажмите Далее.
    4. В текстовом поле "Ключевое слово:" введите ключевое слово для поиска.
    5. В выпадающем списке "Резервные копии:" выберите папку резервирования.
    6. Нажмите кнопку Поиск.
    7. В рамке с правой стороны выберите файлы и папки, которые необходимо восстановить, затем нажмите Далее.
    8. Выберите путь восстановления файлов резервирования. Затем нажмите Готово.

    Совет относительно файлов Outlook, Outlook Express и Windows Mail

         При выборе пути назначения можно выбрать:

    1. Исходный путь
      2.

      При выборе данной опции не придется вручную импортировать почту и адресную книгу, данные ПК будут заменены восстановленными файлами.

    2. Новый путь
      3.

      При выборе данной опции можно восстановить данные по заданному пути, и вручную импортировать почту и адресную книгу из Outlook/Outlook Express/Windows Mail.

Дополнительные настройки

В разделе Опции можно задать расширенные настройки следующим образом.

Фильтр

Используйте данную опцию для точного резервного копирования файлов. Если исключить из списка резервирования файлы с определенным префиксом или расширением, это ускорит процесс резервирования. Выполните следующие действия:

  1. Нажмите Фильтр.
  2. Нажмите Добавить справа от опции "Расширение" или "Prefix".
  3. Введите заданный критерий, например, ".exe" для расширения или "~" для префикса.
  4. Нажмите OK, чтобы закрыть диалоговое окно настройки, затем повторно нажмите OK, чтобы сохранить настройки.

Включить уведомление по электронной почте

Используйте данную функцию для получения подробной информации о резервировании по электронной почте, после завершения запланированного резервирования.

  1. Нажмите Настройка SMTP.
  2. Введите информацию о действующей электронной почте, затем нажмите OK для завершения.

Включить версии файла

Для чего нужна функция версий файлов?

Активация функции версий файлов поможет вести историю версий одного и того же файла.

После завершения первого резервирования все резервные копии файлов будут сохранены в папку ''latest'', в месте резервирования. Начиная со следующего резервирования Data Replicator 3 будет автоматически проверять наличие файла с идентичным именем в папке "latest". При обнаружении такого файла, и если содержимое файлов было изменено, предыдущий файл резервного копирования в папке "latest" будет перемещен в папку "past" как старая версия, в то время как файл с ПК будет зарезервирован в папку "latest", в которой всегда можно найти последние версии файлов резервирования.

Примечание.

Упомянутый выше файл должен быть в той же папке. Поэтому при использовании той же учетной записи для резервирования папки создается предыдущая версия.

Если активирована функция версии файла, и локальные файлы были подвергнуты необратимым или неправильным изменениям, в папке резервирования можно найти архивные файлы и легко их восстановить.

Количество прошлых версий

Здесь можно установить количество прошлых версий файлов резервирования, которые необходимо сохранять, 20 версий как минимум. К именам файлов прошлых версий будет добавляться "v00", "v01" и так далее. Чем больше номер, тем новее версия.

Задайте количество прошлых версий, чтобы определить, сколько предыдущих версий будут храниться в папке резервирования. Выполните следующие действия:

  1. Установите флажок в поле Включить версии файла.
  2. Задайте количество прошлых версий и нажмите Применить, чтобы сохранить настройки.

Точки восстановления

Что такое точки восстановления?

Каждый раз после резервирования появляется папка, имя которой содержит время начала резервирования, содержащая все файлы резервирования. Это и есть точка восстановления.

Файл, который не подвергался изменениям, будет появляться в каждой точке восстановления после каждого резервирования. Однако, фактически на диске занято место, соответствующее размеру только одного файла.

Количество точек восстановления

Можно установить необходимо количество поддерживаемых точек восстановления, максимум 128 точек восстановления. Если количество точек восстановления превысит заданное значение, самые старые точки восстановления в целевой папке резервирования будут автоматически удалены после следующего резервирования.

Поддержка и ограничение
Функция точек восстановления поддерживается только в операционных системах Windows 2000 / Windows XP / Windows 2003 / Windows Vista.

Автоматический запуск при входе

Можно настроить программу Synology Data Replicator 3 на автоматический запуск при входе в Windows. Выполните следующие действия:

  1. Установите флажок в поле Автоматически запускать Data Replicator 3 при запуске Windows.
  2. Нажмите Применить.

Проверка соответствия данных перед синхронизацией

Вы можете выполнить проверку соответствия данных перед переходом Data Replicator в режим синхронизации. Выполните следующие действия:

  1. Установите флажок Проверка соответствия данных перед синхронизацией.
  2. Нажмите Применить.

Примечание.

При выключении данного параметра и закрытии Data Replicator добавленные, удаленные и измененные данные не будут синхронизированы с сервером при следующем запуске Data Replicator.

Сохранять идентичные файлы на целевом сервере

Можно установить в настройках, чтобы файлы в папке "latest" были идентичны файлам на локальном компьютере. Выполните следующие действия:

  1. Установите флажок в поле При удалении локальных файлов одновременно удалите противоположные файлы резервирования.
  2. Нажмите Применить.

Журналы

Максимальный размер файла журнала 512 КБ; когда размер файла журнала достигает предела, самые старые 128 КБ будут удалены.

  • Сохранить журнал
    1. Нажмите Сохранить.
    2. Выберите путь резервирования и нажмите Сохранить для завершения.

  • Очистить журнал
    1. Нажмите Очистить.
    2. Нажмите Да в диалоговом окне подтверждения.
